# app-site-resource

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面可能是静态网页、演示文稿、工具界面或其他前端资源。仓库本身不针对任何特定域名或网站，仅作为页面资源的集中存储与管理。

## 目录说明

```
/
├── index.html          # 仓库首页 / 导航页
├── pages/              # 存放所有归档的 HTML 页面
│   ├── page-1.html
│   ├── page-2.html
│   └── ...
├── assets/             # 公共资源文件（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/` 目录：每个独立 HTML 页面存放于此，文件名应保持唯一且具有描述性。
- `assets/` 目录：存放页面共用的样式、脚本或图片资源，便于统一维护。

## 页面归档说明

- 所有页面均为独立 HTML 文件，可脱离服务器直接运行（如通过 `file://` 协议或本地静态服务器）。
- 页面之间不共享域名或网站上下文，各自拥有完整的 HTML 结构与资源引用。
- 如需引用公共资源，请使用相对路径指向 `assets/` 目录中的文件。

## 维护说明

- 新增页面：在 `pages/` 目录下添加新的 HTML 文件，并确保资源引用路径正确。
- 更新页面：直接修改对应文件，注意保持向后兼容（如链接、样式等）。
- 删除页面：移除相关文件即可，建议同步更新 `index.html` 中的导航链接（如果存在）。
- 资源管理：请勿在 `assets/` 中存放与页面无关的文件，保持目录整洁。

## 使用建议

- 可通过 GitHub Pages 或其他静态托管服务将本仓库部署为可访问的站点，以便在线浏览页面集合。
- 本地预览时，建议使用简单 HTTP 服务器（如 Python 的 `http.server`）以避免跨域资源加载问题。

## 许可

本仓库内容仅供学习与参考使用，具体页面可能包含独立的版权声明，请以实际文件中的说明为准。
